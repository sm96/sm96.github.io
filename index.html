Explanation:

This first visualization aims to answer the following question: How do the minimum concentrations of the three different antibioticss required to prevent growth in vitro (the MIC value) for each type of bacteria compare to each other?

In order to answer this question through a visualization, I chose to use a scatterplot to easily show the MIC values that corresponded to each type of bacteria. The bacteria names were represented in the x-axis while the MIC values were represented in the y-axis in order to allow position visual encoding to easy show MIC values that correspond to each type of bacteria for each antibiotic. The colors represented the types of antibiotics in order to allow users to immediately see which data points belonged to which antibiotic (effectively showing which antibiotics performed better for some bacteria types compared to other bacteria types).

Another visual encoding I used was shape. One of the aspects of the data set was gram staining results for each of the bacteria types. Gram-positive bacteria are more susceptible to antibiotics than gram-negative bacteria. By giving shape to the data points for each of types of bacteria, I facilitated effective communication of which types bacteria are more susceptible to antibiotics compared to other types of bacteria.

Explanation:

The second visualization aims to answer the following question: which of the three antibiotics is the most effective and least effective in preventing growth in vitro for each of the bacteria types?

With this question in mind, I chose to use a divided bar chart to make it easier for users to see proportions (area visual encoding) to compare the performances of the three antibiotics to each other for each individual type of bacteria (with each type of bacteria represented by a bar). With the proportions, it is easy to communicate which of the three antibiotics performed the best for the particular type of bacteria (in this case, the smallest proportion of the bar performs the best because it corresponds to the lowest MIC value).

The other two visual encodings used were color and position. The colors represented the type of antibiotics so the user could easily differentiate the three antibiotic types and compare their values. The height/position of the bars show MIC value totals, indicating which bacteria_types are easier or harder to prevent. The positioning of each bar represents whether or not the gram staining for that particular bacteria type was positive or negative. Bacteria types with positive results were positioned in the positive realm of the graph with a positive scale and bacteria types with negative results were positioned in the negative realm of the graph with a negative scale. This makes it also very easy to see the gram stain results and understand which types bacteria are more susceptible to antibiotics compared to other types of bacteria while taking into account the proportions of each antibiotic for each type of bacteria.

Explanation:

The third visualization aims to answer the following question: how do types of bacteria with positive and negative gram staining results compare?

For this question, I decided to use a scatterplot oriented horizontally (switching the x-axis and y-axis to put a different perspective on the data and focus on allowing easier comparison of size and MIC value for the different data points top to bottom. The main visual encodings for this visualization are color, area, shape, and position. Position basically indicates the performance of antibiotics through their position on the x-axis of MIC values. The shape of the data points indicates the type of antibiotic, making it easier for users to differentiate the types of data points.

For the color encoding, the color corresponds to the gram staining result of the data point. The colors represent which colors would show up for positive and negative results in actual experiments in the real world (violet/dark blue for positive and red for negative) to make it more intuitive for scientists that may be using this visualization. With this, users can see which bacteria types had which gram staining result. For area encoding, the size of the circles represent the total number of types of bacteria that had a positive or negative result. Because there were more types of bacteria that resulted in a negative gram stain, the negative data point circles are much bigger than the the positive data point circles to visually be able to see this information easily and thus easily compare which types bacteria are more susceptible to antibiotics compared to other types of bacteria as well as easily see the general trend in the data of more bacteria with negative gram results.
